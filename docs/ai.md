# AI運用メモ

## 依頼時に伝えると良い情報
- 目的（UI改善、推論高速化、接続安定化 など）
- 対象範囲（backend / frontend / 両方）
- 現在の問題点・期待動作
- 実行環境（OS、Python/Nodeのバージョン、GPU有無）
- 再現手順・エラーログ

## コーディング方針
- UIの操作説明は日本語を基本とする。
- 推論領域は赤枠矩形で視認性を重視する。
- API変更時は `docs/api.md` を必ず更新する。
- コード変更時は関連ドキュメントも必ず更新する。
- 仕様が不明な場合は、先に確認してから実装する。

## 既知の前提
- 推論結果は領域内の検出ラベルを左から連結する仕様。
- 推論領域名は重複不可。
